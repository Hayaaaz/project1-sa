Service: nginx
Host: lab-server1
Author: Haya

------------------------------------------------------------
1. Purpose
------------------------------------------------------------
This runbook provides the steps to safely check, restart, and
verify the nginx web server on lab-server1. All commands use
sudo to follow least‑privilege principles.

------------------------------------------------------------
2. Preconditions
------------------------------------------------------------
- User has sudo privileges.
- nginx is installed and enabled.
- System is Ubuntu Server (26.04 LTS).
- Internet access is not required for restart operations.

------------------------------------------------------------
3. Check nginx Status
------------------------------------------------------------
sudo systemctl status nginx

Expected:
- Active: active (running)
- Enabled: yes

If inactive:
sudo systemctl start nginx

------------------------------------------------------------
4. Restart nginx Safely
------------------------------------------------------------
sudo systemctl restart nginx

Check exit code:
echo $?

0 = success  
Non‑zero = failure (check logs)

------------------------------------------------------------
5. Verify nginx After Restart
------------------------------------------------------------
A. Check service health:
sudo systemctl status nginx

B. Check port 80:
sudo ss -tulpn | grep nginx

C. Test locally:
curl -I http://localhost

Expected:
HTTP/1.1 200 OK

------------------------------------------------------------
6. Check Logs
------------------------------------------------------------
sudo journalctl -u nginx --since "5 minutes ago"

Error logs:
sudo tail -n 50 /var/log/nginx/error.log

Access logs:
sudo tail -n 50 /var/log/nginx/access.log

------------------------------------------------------------
7. Common Fixes
------------------------------------------------------------
A. Syntax errors:
sudo nginx -t

B. Reload without downtime:
sudo systemctl reload nginx

C. Permissions issues:
sudo chown -R www-data:www-data /var/www/html

------------------------------------------------------------
8. Escalation
------------------------------------------------------------
If nginx fails repeatedly:
- Capture logs
- Document symptoms
- Escalate to instructor with:
  * status output
  * error log excerpt
  * steps attempted
